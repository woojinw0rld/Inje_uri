## 전체 구조 개요


src/
├── app/ # Next.js App Router (페이지 + API 라우트)
│ ├── (main)/ # 기존 모바일 페이지 (레이아웃 공유)
│ └── api/ # ★ 새로 추가: 서버 API 엔드포인트
│ ├── auth/ # A 담당: 로그인/회원가입/세션
│ ├── users/ # A 담당: 사용자 정보/프로필
│ ├── keywords/ # A 담당: 키워드 카탈로그
│ ├── recommendations/ # B 담당: 하루 추천/설정
│ ├── interests/ # B 담당: 호감 CRUD/상태전이
│ ├── chat/ # C 담당: 채팅방/메시지
│ ├── feeds/ # D 담당: 셀프소개 피드
│ ├── blocks/ # D 담당: 차단
│ ├── reports/ # D 담당: 신고
│ └── admin/ # D 담당: 운영 (잡, 설정)
│
├── server/ # ★ 새로 추가: 서버 전용 로직 (클라이언트 번들 제외)
│ ├── db/ # Prisma 클라이언트 싱글톤
│ ├── lib/ # 공통 서버 유틸 (응답 포맷, 에러, 인증)
│ ├── repositories/ # DB 접근 계층 (CRUD 추상화)
│ └── services/ # 비즈니스 로직 계층 (규칙/정책)
│
├── components/ # 기존 UI 컴포넌트 (변경 없음)
└── lib/ # 기존 공유 로직 (types, utils, navigation)

## API 구조

### auth


POST /api/auth/register
POST /api/auth/login
POST /api/auth/logout


### users


GET /api/users/me
PATCH /api/users/me
DELETE /api/users/me


### recommendations


GET /api/recommendations/today
POST /api/recommendations/:id/pass
POST /api/recommendations/:id/select


### interests


GET /api/interests
POST /api/interests
POST /api/interests/:id/accept
POST /api/interests/:id/decline


### chat


GET /api/chat/rooms
GET /api/chat/rooms/:id/messages
POST /api/chat/rooms/:id/messages


### feeds


GET /api/feeds
POST /api/feeds
GET /api/feeds/:id
DELETE /api/feeds/:id


### safety


POST /api/blocks
DELETE /api/blocks/:id
POST /api/reports

## 규칙 정리

1. `route.ts`
   - HTTP 처리
   - 인증 처리
   - service 호출

2. `service`
   - 비즈니스 로직
   - 트랜잭션 처리

3. `repository`
   - DB 접근만 담당

4. `server/` 내부 코드
   - 클라이언트 코드 금지

5. Prisma
   - 반드시 `db/index.ts`에서만 생성
